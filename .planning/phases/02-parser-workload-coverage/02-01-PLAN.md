# Plan 02-01: Workload Corpus and Harness Contract

<objective>
Define the Phase 02 workload corpus and wire the execution entrypoints that will consume it.

Purpose: Turn the abstract parser goal into a checked-in supported/excluded fixture matrix that execution can build against deterministically.
Output: Fixture corpus, manifest/schema, workload-source reconciliation artifact, Bazel target wiring, and a repeatable workload command path.
</objective>

<tasks>
<task type="auto">
  <name>Task 1: Check in the supported and excluded workload fixture corpus</name>
  <files>testdata/parser_workload/, examples/bootstrap/</files>
  <action>Create a repo-owned fixture layout for Phase 02 with supported queries, excluded queries, and a checked-in manifest at `testdata/parser_workload/manifest.json`. The manifest must enumerate every fixture file and carry metadata fields for priority, clause tags, expected result kind, expected canonical summary, exclusion reason, and the normalized workload-source identifier used for reconciliation. Keep the corpus representative of the prioritized internal SELECT workload rather than broad SQL parity.</action>
  <verify>
    <automated>bash tools/parser_workload_suite.sh audit --manifest testdata/parser_workload/manifest.json</automated>
    <automated>bash tools/parser_workload_suite.sh list --manifest testdata/parser_workload/manifest.json</automated>
  </verify>
  <done>The repo contains a concrete workload corpus that names which queries Phase 02 must parse and which forms remain intentionally excluded.</done>
</task>

<task type="auto">
  <name>Task 2: Reconcile the checked-in corpus against the real prioritized workload source and record sign-off</name>
  <files>testdata/parser_workload/, tools/, docs/parser_workload_reconciliation.md</files>
  <action>Check in the normalized workload-source snapshot or export metadata required to compare the fixture corpus against the real prioritized internal workload source, then add a deterministic reconciliation command that reports source-to-fixture coverage, any deferred or excluded items, and missing mappings. Record the approved reconciliation result in `docs/parser_workload_reconciliation.md`, including source snapshot identity, mapping counts, unresolved gaps, and reviewer/date sign-off fields.</action>
  <verify>
    <automated>bash tools/parser_workload_suite.sh reconcile --manifest testdata/parser_workload/manifest.json --source testdata/parser_workload/prioritized_internal_workload.json --signoff docs/parser_workload_reconciliation.md</automated>
    <automated>bash tools/parser_workload_suite.sh audit-signoff --manifest testdata/parser_workload/manifest.json --source testdata/parser_workload/prioritized_internal_workload.json --signoff docs/parser_workload_reconciliation.md</automated>
  </verify>
  <done>Phase 02 has executable evidence that the checked-in fixture corpus is still aligned with the real prioritized workload source, and the sign-off artifact makes any approved omissions explicit.</done>
</task>

<task type="auto">
  <name>Task 3: Add Bazel wiring for a dedicated workload-summary validation target</name>
  <files>examples/bootstrap/BUILD.bazel</files>
