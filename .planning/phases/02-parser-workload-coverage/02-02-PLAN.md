# Plan 02-02: Canonical Summary Runner and Fixture-Driven Parser Fixes

<objective>
Implement the executable workload-summary contract and close parser gaps revealed by supported fixtures.

Purpose: Convert the fixture corpus from Plan 02-01 into exact parse-success and determinism proof for the in-scope SELECT workload.
Output: Canonical workload-summary binary, full-corpus deterministic comparison harness, and any required narrow parser-layer fixes.
</objective>

<tasks>
<task type="auto">
  <name>Task 1: Implement a dedicated workload-summary runner with stable structural output</name>
  <files>examples/bootstrap/, examples/bootstrap/BUILD.bazel</files>
  <action>Add or fork a validation binary that parses a query or a manifest-listed fixture and emits a compact canonical summary covering query kind, clause presence, source kinds, join modifiers, set operations, and normalized limit/limit-by data. The runner must fail if any supported fixture leaves trailing unconsumed tokens. Support an explicit contract of `--manifest testdata/parser_workload/manifest.json --mode <supported|excluded|quick|repeat>` where `repeat` walks the full supported fixture set, so verify commands never depend on implicit cwd discovery. Keep it intentionally narrower than a full AST or Phase 03 IR.</action>
  <verify>
    <automated>bazel --batch --output_user_root=/tmp/bazel-root run //examples/bootstrap:parser_workload_summary -- --manifest testdata/parser_workload/manifest.json --mode quick</automated>
  </verify>
  <done>The phase has a machine-comparable parser output contract strong enough to distinguish workload shapes, reject trailing unconsumed tokens, and remain phase-local and parser-only.</done>
</task>

<task type="auto">
  <name>Task 2: Implement exact summary comparison and repeat-mode determinism checks</name>
  <files>tools/, examples/bootstrap/, testdata/parser_workload/</files>
  <action>Extend the workload harness so supported fixtures compare actual vs expected summaries exactly, assert full token consumption for every supported fixture, and run repeat mode over every supported fixture multiple times in one command, failing on any output drift for any accepted workload query.</action>
  <verify>
    <automated>bash tools/parser_workload_suite.sh supported --manifest testdata/parser_workload/manifest.json</automated>
    <automated>bash tools/parser_workload_suite.sh repeat --manifest testdata/parser_workload/manifest.json</automated>
  </verify>
  <done>Supported fixtures now enforce exact canonical output, full token consumption, and full-corpus determinism rather than parse-success alone.</done>
</task>

  <name>Task 3: Patch only the parser layers exposed by failing supported fixtures</name>
